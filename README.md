# 🦷 Dental ClinicCare

Dental ClinicCare is a comprehensive dental clinic management system developed to automate and streamline the daily operations of a dental clinic. The system provides role-based access control, appointment scheduling, patient record management, treatment tracking, and reporting functionalities through a centralized and user-friendly platform.

The application helps clinics improve efficiency, reduce paperwork, maintain accurate patient records, and enhance the overall quality of healthcare services.

## ✨ Key Features

### 🔐 Authentication & Authorization

* Secure login system.
* Role-based access control.
* Different permissions for Admin, Receptionist, and Doctor users.

### 👨‍💼 Admin Module

* Manage users and system access.
* Monitor clinic operations.
* Generate and review reports.
* Manage dentists and clinic data.

### 🧑‍💻 Receptionist Module

* Register new patients.
* Schedule, update, and cancel appointments.
* Manage patient information.
* Handle daily appointment operations.

### 👨‍⚕️ Doctor Module

* View assigned appointments.
* Access patient medical records.
* Record diagnoses and treatments.
* Update treatment history.

### 📋 Patient Management

* Store patient personal information.
* Maintain medical history records.
* Search and update patient data efficiently.

### 📅 Appointment Management

* Schedule appointments.
* Track appointment status.
* Organize doctor schedules.

### 📊 Reports

* Generate clinic reports.
* Monitor appointments and patient activities.
* Support administrative decision-making through organized data.

## 🛠️ Technologies Used

* C#
* .NET Framework
* SQL Server
* ADO.NET
* Windows Forms

## 🎯 Project Objectives

* Digitize dental clinic operations.
* Improve patient data management.
* Enhance appointment scheduling efficiency.
* Maintain secure and accurate medical records.
* Provide role-based access to ensure data security.
* Generate useful reports for clinic administration.

## 🔄 System Workflow

1. Users log in according to their assigned roles.
2. Receptionists register patients and schedule appointments.
3. Doctors access patient records and update treatment information.
4. Administrators manage users, monitor activities, and generate reports.
5. All data is securely stored and managed through a centralized SQL Server database.

## 👥 User Roles

* **Admin**
* **Receptionist**
* **Doctor**

Each role has dedicated functionalities and permissions to ensure secure and efficient clinic management.

## 📚 Educational Purpose

This project was developed as an academic software engineering project to demonstrate practical skills in desktop application development, database design, authentication systems, role-based access control, and SQL Server integration.
